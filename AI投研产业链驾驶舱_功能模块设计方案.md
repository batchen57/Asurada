# AI 投研产业链驾驶舱功能模块设计方案

## 一、系统定位

本系统不是普通的投研展示看板，而是一个：

> **研报驱动的产业链深度研究与估值决策系统**

核心目标是基于大量研报、行业数据、公司数据、行情数据，通过 AI 结构化提炼、交叉验证、指标建模和可视化看板，形成对一个板块的中长期投资判断。

系统需要解决以下核心问题：

1. 哪些板块已经分析，哪些还没有分析？
2. 某个板块的产业链、成本结构、技术壁垒、国产化率、替代风险是什么？
3. 每个核心零部件或细分环节谁是龙头，谁在追赶，差距多大？
4. 赛道是否安全？壁垒是否真实？是否会被替代？
5. 对应上市公司的估值是否合理，还需要多久消化？

## 二、总体功能模块架构

系统建议拆分为 12 个核心模块：

```text
投研板块管理
   ↓
研报/数据源管理
   ↓
AI 研报解析与知识库
   ↓
产业链总览
   ↓
成本结构与降本路径
   ↓
核心模块分析检测器
   ↓
技术路线/工艺路线对比
   ↓
竞争格局与公司对标
   ↓
替代风险与壁垒评估
   ↓
估值全景与行情财务数据
   ↓
投研结论与风险提示
   ↓
证据溯源与可信度管理
```

## 三、一级功能模块设计

### 1. 投研板块管理模块

这是系统入口，用来管理所有待研究行业和赛道。

#### 核心功能

管理每一个投研板块，例如：

- 人形机器人
- 半导体设备
- 工业母机
- 固态电池
- 光模块
- AI 算力
- 低空经济
- 储能
- 冷链设备
- 机器人减速器
- 传感器
- 电机

#### 页面字段

| 字段 | 说明 |
|---|---|
| 板块名称 | 如人形机器人、半导体设备 |
| 所属大类 | 制造、TMT、新能源、医药等 |
| 研究状态 | 未开始、资料收集中、AI解析中、待人工确认、已分析、持续跟踪 |
| 研报数量 | 当前已接入多少篇研报 |
| 数据更新时间 | 最近一次数据刷新时间 |
| 核心结论状态 | 未生成、已生成、需复核 |
| 估值覆盖率 | 是否已经覆盖主要上市公司 |
| 风险等级 | 低、中、高 |
| 机会等级 | S/A/B/C |
| 负责人 | 研究负责人 |
| 标签 | 高景气、国产替代、技术突破、周期底部等 |

#### 板块状态流转

```text
新建板块
  ↓
配置研究框架
  ↓
导入研报/数据源
  ↓
AI 解析
  ↓
指标抽取
  ↓
交叉验证
  ↓
生成产业链总览
  ↓
生成各子模块检测器
  ↓
接入行情和财务数据
  ↓
生成估值全景
  ↓
形成投研结论
  ↓
进入持续跟踪
```

#### 实现逻辑

每个板块本质上是一个 `ResearchProject`。

它下面挂载：

- 研报集合
- 产业链节点
- 成本结构
- 技术路线
- 公司池
- 风险模型
- 估值模型
- AI 结论
- 证据链

### 2. 研报与数据源管理模块

这是整个系统的底层能力。

你的方案里最重要的一点是：

> 结论必须来自研报，而不是 AI 随机生成。

所以这个模块必须非常扎实。

#### 数据源类型

| 数据源 | 用途 |
|---|---|
| 研报 PDF | 产业链、成本、技术、竞争格局、风险来源 |
| Asoft Data 接口 | 批量获取研报、行业数据 |
| 行情 API | 股价、市值、PE、PB、PEG |
| 财务 API | 营收、利润、增速、ROE、毛利率 |
| 公司公告 | 产能、订单、客户、技术进展 |
| 人工补充数据 | 研究员手工录入校准数据 |

#### 研报管理功能

支持：

- 批量导入研报
- 自动识别研报标题、机构、发布日期、行业、公司
- 自动归属到对应板块
- 研报去重
- 研报质量评分
- 研报内容切片
- 表格抽取
- 图表 OCR
- 关键数据抽取
- 引用来源定位
- 研报版本管理

#### 研报状态

```text
已导入
  ↓
文本解析完成
  ↓
章节识别完成
  ↓
表格抽取完成
  ↓
AI 提炼完成
  ↓
指标入库完成
  ↓
人工复核完成
```

#### 关键设计

每一个 AI 生成的结论，都要绑定：

```text
结论 → 指标 → 原始研报 → 页码 → 原文片段 → 置信度
```

示例：

```text
结论：谐波减速器国产化率提升明显

来源：
- XX证券《人形机器人产业链深度报告》P23
- XX证券《机器人核心零部件跟踪》P17
- XX证券《绿的谐波深度报告》P11

置信度：高
```

### 3. AI 研报解析与知识库模块

这是系统的智能中枢。

#### AI 处理目标

AI 不只是总结研报，而是要做结构化提炼。

它要把非结构化研报转成这些结构化对象：

```text
行业
产业链节点
成本项目
零部件
核心公司
技术路线
国产化率
市场份额
壁垒类型
替代风险
降本路径
估值指标
风险点
核心结论
```

#### AI 解析流程

```text
研报原文
  ↓
文本清洗
  ↓
章节识别
  ↓
表格/图片解析
  ↓
内容切片 Chunk
  ↓
Embedding 入向量库
  ↓
实体识别
  ↓
指标抽取
  ↓
多研报交叉验证
  ↓
结构化入库
  ↓
生成看板数据
```

#### 关键 AI 任务

##### 任务 1：产业链实体识别

抽取：

- 上游材料
- 核心零部件
- 中游制造
- 下游应用
- 代表公司
- 成本占比
- 国产化率
- 技术壁垒
- 竞争格局

##### 任务 2：成本结构抽取

示例结构：

```json
{
  "module": "谐波减速器",
  "cost_ratio": "35%",
  "unit_cost": "xxx 元",
  "cost_decline_path": "2024-2027 CAGR -15%",
  "evidence": "研报来源"
}
```

##### 任务 3：竞争格局抽取

示例结构：

```json
{
  "segment": "谐波减速器",
  "global_leader": "哈默纳科",
  "domestic_leader": "绿的谐波",
  "global_share": "12%",
  "domestic_share": "60%",
  "followers": ["来福谐波", "丰立智能", "中大力德"],
  "gap_to_leader": "精度、寿命、一致性仍有差距"
}
```

##### 任务 4：壁垒与替代风险抽取

示例结构：

```json
{
  "segment": "六维力传感器",
  "barrier_score": 75,
  "substitution_risk": "中高",
  "replacement_window": "3-5年",
  "risk_reason": "3D视觉、算法补偿、低成本方案可能部分替代"
}
```

##### 任务 5：估值数据生成

结合实时行情和财务 API，生成：

- PE
- PB
- PEG
- 市值
- 净利润
- 收入增速
- 利润增速
- 当前估值分位
- 估值消化时间
- 目标估值距离

### 4. 产业链总览模块

这是选择某个板块后的首页。

#### 页面目标

让用户一眼看懂：

- 这个产业链由哪些环节组成
- 哪些环节成本最高
- 哪些环节国产化率最低
- 哪些环节技术壁垒最高
- 哪些环节替代风险最大
- 哪些环节投资价值最高

#### 页面结构

建议分成 7 个区域：

```text
顶部：板块核心结论卡片
中部：产业链全景图
左侧：成本结构图
右侧：国产化率/卡脖子环节
下方：核心公司矩阵
下方：替代风险雷达图
底部：证据来源与可信度
```

#### 核心指标

| 指标 | 说明 |
|---|---|
| 总成本 | 一台设备/系统整体成本 |
| 成本构成 | 各零部件成本占比 |
| 国产化率 | 国内厂商替代程度 |
| 技术壁垒 | 技术难度评分 |
| 供应链安全度 | 是否依赖海外 |
| 替代风险 | 是否可能被其他路线替代 |
| 降本空间 | 未来几年成本下降幅度 |
| 投资吸引力 | 综合评分 |

#### 产业链节点卡片

每个节点建议展示：

```text
节点名称：谐波减速器
成本占比：35%
国产化率：中
技术壁垒：高
全球龙头：哈默纳科
国内龙头：绿的谐波
替代风险：低
投资价值：高
状态：重点关注
```

点击节点后进入对应的“模块分析检测器”。

### 5. 成本结构与降本路径模块

这是非常核心的一页。

#### 页面目标

回答三个问题：

1. 当前一套系统成本是多少？
2. 每个模块成本占比是多少？
3. 未来如何降本，目标价是多少？

#### 当前成本拆解

| 模块 | 当前成本 | 成本占比 | 供应商 | 国产化率 | 降本空间 |
|---|---:|---:|---|---|---|
| 减速器 | xxx | 35% | 哈默纳科/绿的谐波 | 中 | 高 |
| 电机 | xxx | 20% | 汇川/鸣志等 | 较高 | 中 |
| 传感器 | xxx | 10% | ATI/国内厂商 | 低 | 中 |
| 控制器 | xxx | 15% | 国内/海外 | 中 | 中 |
| 结构件 | xxx | 20% | 国内 | 高 | 低 |

#### 降本路径

```text
2024 当前成本
  ↓ -15%
2025 规模化降本
  ↓ -20%
2026 国产替代降本
  ↓ -10%
2027 工艺成熟降本
  ↓
目标成本
```

#### 目标价格对比

| 主体 | 目标价格 | 时间窗口 | 依据 |
|---|---:|---|---|
| 特斯拉 | 2万美元 | 中期 | 研报/公开资料 |
| 国产厂商 | xxx | 中期 | 研报预测 |
| 当前行业平均 | xxx | 当前 | 研报统计 |

#### 实现逻辑

成本数据来源分为三类：

```text
研报抽取成本数据
人工修正成本数据
模型推算成本数据
```

系统要区分数据类型：

| 类型 | 可信度 |
|---|---|
| 多篇研报一致提及 | 高 |
| 单篇研报提及 | 中 |
| AI 推算 | 低 |
| 人工录入 | 高，但需要标注录入人 |

每个成本数值后面都要能点开证据。

### 6. 核心模块分析检测器

这是系统最有价值的分析单元。

每个产业链节点都可以生成一个“检测器”。

例如：

- 谐波减速器检测器
- 行星滚柱丝杠检测器
- 电机检测器
- 六维力传感器检测器
- 3D 视觉检测器
- 控制器检测器
- 线性执行器检测器
- 热管理检测器

#### 检测器页面结构

每个检测器建议包含 9 个区域：

```text
1. 模块基本信息
2. 技术壁垒评分
3. 成本占比
4. 国产化率
5. 竞争格局
6. 龙头对标
7. 技术路线对比
8. 替代风险
9. 投资结论
```

#### 检测器核心字段

| 字段 | 示例 |
|---|---|
| 模块名称 | 谐波减速器 |
| 所属产业链位置 | 核心零部件 |
| 成本占比 | 高 |
| 技术壁垒 | 高 |
| 主要壁垒 | 精度、寿命、一致性、加工设备 |
| 全球龙头 | 哈默纳科 |
| 国内龙头 | 绿的谐波 |
| 国产化率 | 中 |
| 替代风险 | 低 |
| 未来窗口期 | 5-10年 |
| 重点公司 | 绿的谐波、中大力德、丰立智能 |
| 投资等级 | A |

#### 技术壁垒评分模型

| 维度 | 权重 | 说明 |
|---|---:|---|
| 材料壁垒 | 15% | 是否依赖特殊材料 |
| 加工工艺 | 25% | 精密制造难度 |
| 设备壁垒 | 15% | 是否需要高端设备 |
| Know-how | 20% | 是否依赖长期经验 |
| 良率壁垒 | 15% | 量产一致性难度 |
| 客户验证 | 10% | 进入客户供应链难度 |

计算公式：

```text
技术壁垒分 = Σ 维度得分 × 权重
```

示例：

```text
谐波减速器：88分
六维力传感器：75分
普通结构件：40分
```

### 7. 技术路线与工艺路线对比模块

该模块用于分析“路线之争”和“工艺路线之争”。

#### 适用场景

例如：

- 谐波减速器 vs RV 减速器
- 空心杯电机 vs 无框力矩电机
- 六维力传感器 vs 3D 视觉替代
- 滚珠丝杠 vs 行星滚柱丝杠
- 传统传感器方案 vs 视觉 + 算法方案

#### 页面设计

| 维度 | 路线 A | 路线 B | 结论 |
|---|---|---|---|
| 成本 | 中 | 高 | A 更低 |
| 精度 | 高 | 更高 | B 更优 |
| 寿命 | 中 | 高 | B 更优 |
| 量产难度 | 低 | 高 | A 更成熟 |
| 国产化率 | 高 | 低 | A 更安全 |
| 替代风险 | 中 | 低 | B 更稳 |
| 适用场景 | 中低端 | 高端 | 分层共存 |

#### 实现逻辑

AI 从研报中抽取路线差异，结构化成：

```json
{
  "route_a": "谐波减速器",
  "route_b": "RV减速器",
  "compare_dimensions": [
    {
      "dimension": "精度",
      "route_a_score": 85,
      "route_b_score": 78,
      "winner": "route_a",
      "evidence": "研报出处"
    }
  ],
  "final_judgement": "高精度轻量场景谐波更优，重载场景 RV 更优"
}
```

### 8. 竞争格局与公司对标模块

这个模块解决：

> 龙头是谁？国内企业追到什么水平？投资标的谁最值得看？

#### 页面结构

建议分成 4 层：

```text
全球格局
国内格局
公司能力对比
龙头差距分析
```

#### 核心表格

| 公司 | 地位 | 全球份额 | 国内份额 | 技术水平 | 产能 | 客户 | 毛利率 | 估值 |
|---|---|---:|---:|---|---|---|---:|---:|
| 哈默纳科 | 全球龙头 | 高 | - | 标杆 | 高 | 国际大厂 | 高 | - |
| 绿的谐波 | 国内龙头 | 12% | 60% | 接近 | 中高 | 国内客户 | 高 | 高 |
| 中大力德 | 追赶者 | 低 | 中 | 追赶 | 中 | 国内 | 中 | 中 |
| 丰立智能 | 追赶者 | 低 | 低 | 早期 | 低 | 国内 | 中 | 高 |

#### 龙头差距分析

| 维度 | 国内龙头 | 海外龙头 | 差距 |
|---|---|---|---|
| 精度 | 接近 | 标杆 | 小 |
| 寿命 | 有差距 | 标杆 | 中 |
| 稳定性 | 有差距 | 标杆 | 中 |
| 批量一致性 | 有差距 | 标杆 | 大 |
| 客户认证 | 追赶中 | 完成 | 中 |
| 成本 | 更低 | 较高 | 国内优势 |

#### 实现逻辑

AI 从研报中提炼每家公司的能力标签：

```text
公司 → 产品 → 技术参数 → 产能 → 客户 → 市占率 → 估值 → 风险
```

再统一映射成评分：

```text
公司综合竞争力 = 技术分 × 30%
              + 产能分 × 20%
              + 客户分 × 20%
              + 成本分 × 15%
              + 财务分 × 15%
```

### 9. 替代风险与赛道安全性模块

这是中长期判断里非常重要的一层。

#### 页面目标

判断一个细分赛道：

- 壁垒是真壁垒还是伪壁垒？
- 是否会被新技术路线替代？
- 投资窗口期还有多久？
- 这个环节是否值得长期配置？

#### 替代风险评估模型

| 维度 | 说明 |
|---|---|
| 当前不可替代性 | 现阶段是否必须使用 |
| 技术成熟度 | 替代方案是否成熟 |
| 成本优势 | 替代方案是否更便宜 |
| 性能优势 | 替代方案是否更强 |
| 产业推动力 | 大厂是否推动替代 |
| 客户切换成本 | 下游是否愿意换 |
| 时间窗口 | 替代发生需要多久 |

#### 风险等级

```text
低风险：5-10年内不易替代
中风险：3-5年存在替代压力
高风险：1-3年可能被技术路线冲击
```

#### 示例

| 模块 | 壁垒 | 替代风险 | 时间窗口 | 结论 |
|---|---|---|---|---|
| 谐波减速器 | 高 | 低 | 5-10年 | 暂时安全 |
| 六维力传感器 | 中高 | 中高 | 3-5年 | 需要警惕视觉替代 |
| 普通结构件 | 低 | 中 | 2-3年 | 投资价值一般 |
| 3D视觉 | 中 | 低 | 5年+ | 可能受益于替代趋势 |

#### 输出结论

系统最终应该给出：

```text
最安全赛道
最危险赛道
最容易被替代模块
最具长期壁垒模块
短期弹性最大模块
中长期确定性最高模块
```

### 10. 估值全景模块

这是从产业研究落到投资决策的关键模块。

#### 数据来源

接入：

- 行情 API
- 财务 API
- 公司公告
- 预测数据
- 研究员手工假设

#### 核心指标

| 指标 | 说明 |
|---|---|
| 当前股价 | 实时行情 |
| 市值 | 实时计算 |
| PE TTM | 当前估值 |
| Forward PE | 预测估值 |
| PB | 资产估值 |
| PEG | 增长估值 |
| 收入增速 | 景气度 |
| 利润增速 | 业绩弹性 |
| ROE | 质量 |
| 毛利率 | 壁垒 |
| 估值消化时间 | 当前估值需要几年业绩消化 |
| 目标市值空间 | 目标价距离 |

#### 估值消化时间逻辑

你提到的逻辑是：

> 假设 PE 回到 30 倍，按照当前利润增速，看需要几年消化。

可以这样设计：

```text
当前市值 ÷ 第 N 年预测净利润 ≤ 目标 PE
```

示例：

```text
当前市值 = 300 亿
当前净利润 = 5 亿
当前 PE = 60
利润年增速 = 30%
目标 PE = 30
```

逐年推算：

```text
第 1 年利润 = 6.5 亿，PE = 46.1
第 2 年利润 = 8.45 亿，PE = 35.5
第 3 年利润 = 10.98 亿，PE = 27.3
```

结论：

```text
估值消化时间约 3 年
```

```text
下方：估值消化时间
底部：风险提示
```

### 11. 投研结论与风险提示模块

这个模块把前面所有内容收敛成最终决策。

#### 输出形式

建议形成三类结论。

##### 板块级结论

```text
该板块中长期确定性：高
核心驱动：国产替代 + 成本下降 + 下游放量
核心风险：需求不及预期、技术路线变化、估值过高
投资阶段：从主题炒作进入基本面验证期
```

##### 产业链环节结论

```text
最值得关注环节：
1. 谐波减速器
2. 行星滚柱丝杠
3. 空心杯电机

需要谨慎环节：
1. 普通结构件
2. 低价值传感器
3. 易被算法替代的感知模块
```

##### 公司级结论

```text
重点跟踪：
- 龙头型：技术壁垒高，估值偏贵
- 弹性型：份额小但成长快
- 低估型：估值消化时间短
- 风险型：估值高但壁垒不足
```

#### 风险提示分类

| 风险类型 | 说明 |
|---|---|
| 技术路线风险 | 被其他方案替代 |
| 估值风险 | 股价已提前反映长期预期 |
| 业绩风险 | 订单兑现慢 |
| 竞争风险 | 新进入者导致毛利下降 |
| 政策风险 | 政策变化影响需求 |
| 供应链风险 | 核心材料或设备受限 |
| 研报数据风险 | 多篇研报数据存在冲突 |

---

### 12. 证据溯源与可信度管理模块

这个模块是防 AI 幻觉的关键。

#### 每条结论必须支持点击溯源

例如页面上显示：

```text
谐波减速器未来 5-10 年替代风险较低
```

点击后展开：

```text
证据来源：
1. XX证券《人形机器人核心零部件报告》P18
2. XX证券《减速器行业深度》P12
3. XX证券《机器人产业链拆解》P31

一致性：高
冲突数据：无
AI 置信度：87%
人工复核：已确认
```

#### 可信度评分

```text
可信度 = 来源数量 × 30%
       + 来源质量 × 25%
       + 多源一致性 × 25%
       + 数据新鲜度 × 10%
       + 人工复核 × 10%
```

#### 证据状态

| 状态 | 含义 |
|---|---|
| 高可信 | 多篇研报一致，且有明确数据 |
| 中可信 | 有研报支持，但来源较少 |
| 低可信 | AI 推断为主，需要人工确认 |
| 冲突 | 多篇研报说法不一致 |
| 过期 | 数据较老，需要更新 |

```text

  ↓

  ↓

  ↓

  ↓
```

## 十一、关键设计原则

### 1. 结论必须结构化

不要只让 AI 总结，要让 AI 抽取成：

```text
成本、份额、壁垒、风险、估值、证据
```

### 2. 数据必须可溯源

每一个数字都要能看到来源。

```text
数据 → 研报 → 页码 → 原文
```

### 3. AI 必须被方法论约束

不同板块要有不同研究模板。

例如：

```text
人形机器人看零部件
半导体设备看国产替代
新能源看成本曲线
医药看临床进展
AI 算力看供需和资本开支
```

### 4. 看板不是展示，而是决策系统

最终要输出：

```text
哪个环节值得看？
哪个环节有风险？
哪个公司估值合理？
哪个公司只是主题炒作？
```

### 5. 研报库是护城河

这个系统最核心的资产不是代码，而是：

```text
高质量研报库 + 结构化投研方法论 + 证据链
```

## 十二、推荐最终产品形态

这个产品可以定义为：

> **AI 投研产业链驾驶舱**

核心体验是：

```text
选择一个板块
  ↓
看到产业链全景
  ↓
点击核心零部件
  ↓
进入模块检测器
  ↓
查看技术壁垒、竞争格局、国产替代、替代风险
  ↓
跳转公司估值全景
  ↓
生成投资结论
  ↓
```
